# MarsAI 命令手册

本文档详细说明 MarsAI QQ 机器人的所有可用命令。这些命令可用于管理 AI 对话、控制模型行为和处理会话历史。

## 快速命令参考

| 命令 | 描述 | 参数 | 权限 |
|------|------|------|------|
| `/help` | 显示所有可用命令 | 无 | 所有用户 |
| `/clear` | 清除当前会话历史 | 无 | 所有用户 |
| `/reset` | 重置对话（同 `/clear`） | 无 | 所有用户 |
| `/prompt` | 显示、更新系统prompt或切换预设 | `[text|数字|list]` | 所有用户 |
| `/model` | 显示或切换 AI 模型 | `[name]` | 所有用户 |
| `/memory` | 管理长期记忆 | `[clear]` | 所有用户 |
| `/reminder` | 设置和管理提醒 | `[list|cancel|help]` | 所有用户 |
| `/summary` | 总结当前对话内容 | 无 | 所有用户 |
| `/history` | 查看最近对话历史 | `[n]` | 所有用户 |
| `/status` | 显示当前系统状态 | 无 | 所有用户 |
| `/temperature` | 查看或设置采样温度，决定回答随机程度 | `[value]` | 所有用户 |

## 命令详解

### `/help`
显示所有可用命令及其简要说明。

[新增] `/temperature` 命令可以在对话中实时调整 AI 生成回答时使用的温度值，范围 0 到 1，值越低输出越确定，值越高输出越随机。

# `/temperature`
用于查看或设置当前采样温度。

**语法：**
```
/temperature           # 显示当前温度值
/temperature <0-2数字>  # 设置新的温度
```

**示例：**
```
用户：/temperature
AI：当前温度：0.7

用户：/temperature 0.3
AI：已设置温度为 0.3（0-2之间）
```

### `/clear`
清除当前会话的所有历史记录。这个命令不会影响系统 prompt 或模型设置，只会清空对话历史。

**语法：**
```
/clear
```

**示例：**
```
用户：/clear
AI：已清除当前会话历史
```

**注意：**
- 清除后无法恢复历史记录
- 每个会话（私聊或群聊）的历史是独立的
- 清除后，AI 将开始一个全新的对话

---

### `/reset`
功能与 `/clear` 相同，提供另一种方式重置对话。

**语法：**
```
/reset
```

**示例：**
```
用户：/reset
AI：对话已重置
```

### `/prompt`
显示当前系统 prompt、更新为新的 prompt 或切换预设 prompt。**重要：更新 prompt 时会自动清除当前会话历史**，以避免旧对话干扰新的角色设定。

每个预设同时带有默认的采样温度，使用 `/prompt list` 时一并显示，可通过 `/temperature` 命令手动调整。

**语法：**
```
/prompt                    # 显示当前 prompt 预览（/prompt list 还会显示预设温度）
/prompt <new_prompt_text>  # 更新 prompt 并清除历史
/prompt <数字>             # 切换到对应数字的预设 prompt
/prompt list               # 列出所有可用预设
```

**示例：**
```
# 查看当前 prompt
用户：/prompt
AI：当前prompt预览（319字符）：
你是一位深度精通《Minecraft》（我的世界）全版本、全机制的资深玩家...
使用 /prompt <新prompt> 来更新，或使用 /prompt <数字> 切换预设，/prompt list 列出预设

# 更新 prompt
用户：/prompt 你是一个专业的 Python 编程助手，精通各种 Python 库和框架。
AI：系统prompt已更新，当前会话历史已清除

# 切换预设 prompt
用户：/prompt 1
AI：已切换到预设 1，当前会话历史已清除

# 列出预设
用户：/prompt list
AI：可用预设：
1: 你是一位深度精通《Minecraft》（我的世界）全版本... (temp=1.3)
2: 你是一个乐于助人的AI助手，请用友好、简洁的方式... (temp=1.0)
3: 你是一个编程专家，擅长Python、JavaScript、Java... (temp=0.7)
```

**注意：**
- 更新 prompt 后，之前的对话历史会被自动清除
- Prompt 最大长度取决于使用的 AI 模型
- 修改 prompt 会改变 AI 的行为和角色设定
- Prompt 保存在 `session/bot_config.json` 中，重启后仍有效
- 预设 prompt 也保存在配置文件中，可手动编辑添加更多预设

### `/model`
显示当前使用的 AI 模型或切换到其他可用模型。

**可用模型：**
- **云端模型**：
  - `DeepSeek Chat` (deepseek-chat) - 默认模型
- **本地模型**（需安装 Ollama）：
  - `gemma3:12b` (gemma3:12b)
  - `gpt-oss:20b` (gpt-oss:20b)
  - `geem3` (geem3)

**语法：**
```
/model                    # 显示当前模型
/model <model_name>      # 切换到指定模型
```

**示例：**
```
# 查看当前模型
用户：/model
AI：当前模型：DeepSeek Chat

# 切换模型
用户：/model gemma3:12b
AI：已切换模型到：gemma3:12b
```

**注意：**
- 切换模型不会清除对话历史
- 本地模型需要安装并运行 Ollama
- 云端模型需要配置 `DEEPSEEK_API_KEY` 环境变量
- 模型切换后，新的回复将使用新模型生成

### `/summary`
使用 AI 分析并总结当前会话的对话内容。提取关键信息、讨论主题和结论。

**语法：**
```
/summary
```

**示例：**
```
用户：/summary
AI：对话总结：
本次对话主要讨论了 Minecraft 的红石电路设计。用户询问了自动农场的工作原理，
AI 详细解释了活塞、红石火把和比较器的配合使用。对话还涉及了不同版本（Java版
和基岩版）的差异。最后用户得到了一个可用的自动甘蔗农场设计方案。
```

**注意：**
- 总结内容由 AI 生成，可能不完美
- 需要至少有一些对话历史才能总结
- 总结会消耗一次 API 调用
- 总结提示是独立的，不会影响系统 prompt

### `/history`
查看最近的对话历史记录。默认显示最近 10 条消息，可指定数量。

**语法：**
```
/history          # 显示最近10条消息
/history <n>      # 显示最近n条消息（1-50）
```

**示例：**
```
# 查看最近5条消息
用户：/history 5
AI：最近 5 条消息：
1. [用户] 如何建造自动农场？
2. [AI] 自动农场需要红石电路控制...
3. [用户] 需要哪些材料？
4. [AI] 主要需要：活塞、红石火把...
5. [用户] 谢谢，我明白了

# 默认查看最近10条
用户：/history
AI：最近 10 条消息：
...
```

**注意：**
- 最多显示 50 条历史记录
- 消息内容会被截断（前100字符）
- 显示格式：`[用户]` 表示用户消息，`[AI]` 表示 AI 回复
- 历史记录存储在 `session/user_sessions.json` 中

**语法：**

```

**示例：**
```

**显示的信息包括：**
- 当前使用的 AI 模型
- 系统 prompt 的字符数
- 当前会话的历史消息数量

### `/memory`
管理长期记忆功能。长期记忆是 AI 对重要对话内容的总结，自动存储并可在后续对话中引用。

**语法：**
```
/memory           # 显示长期记忆列表
/memory clear     # 清除当前会话的长期记忆
```

**示例：**
```
# 查看长期记忆
用户：/memory
AI：长期记忆（共3条）：
1. [2025-02-28 10:30:00] 用户询问Python异步编程的基础概念
2. [2025-02-28 14:45:00] 讨论了Minecraft红石电路的自动农场设计
3. [2025-02-28 16:20:00] 用户请求帮助调试JavaScript代码

# 清除长期记忆
用户：/memory clear
AI：已清除长期记忆
```

**注意：**
- 长期记忆由 AI 自动总结生成，当对话达到一定长度时会触发自动总结
- 每个会话（私聊或群聊）有独立的长期记忆存储
- 长期记忆存储在 `session/long_term_memory.json` 文件中
- 清除后无法恢复，请谨慎操作

### `/reminder`
智能提醒功能，使用自然语言解析设置提醒，支持重复提醒和多种发送渠道。

**语法：**
```
/reminder                    # 显示提醒功能帮助
/reminder list              # 列出所有提醒（包括已发送和待处理）
/reminder cancel <id>       # 取消指定ID的提醒
/reminder clear             # 清除所有待处理提醒
/reminder help              # 显示详细帮助
/reminder test <id>         # 测试触发指定提醒（仅测试用）
/reminder debug             # 显示调度器调试信息
```

**自然语言设置提醒：**
除了命令外，可以直接用自然语言设置提醒，AI会自动解析：
- "明天下午3点提醒我开会"
- "30分钟后提醒我喝水"
- "每天上午9点提醒我打卡"
- "每周一提醒我提交周报"

**示例：**
```
# 设置提醒
用户：明天下午3点提醒我开会
AI：已设置提醒：明天下午3点提醒您开会

# 列出提醒
用户：/reminder list
AI：您的提醒列表：
⏰ 待处理提醒（2个）：
1. ID: user_123456789_20250228150000
   时间: 2025-02-28 15:00
   内容: 开会
   重复: 单次
   渠道: 原渠道
   状态: 待处理

# 取消提醒
用户：/reminder cancel user_123456789_20250228150000
AI：已取消提醒 user_123456789_20250228150000
```

**提醒属性：**
- **时间**：支持绝对时间（"2025-02-28 15:00"）和相对时间（"30分钟后"）
- **重复规则**：单次（none）、每天（daily）、每周（weekly）、每月（monthly）、自定义cron表达式
- **发送渠道**：原渠道（current）、私聊（private）、群聊（group）
- **状态**：待处理（pending）、已发送（sent）、已过期（expired）、已取消（cancelled）、失败（failed）

**文件存储：**
- 提醒数据保存在 `session/reminders.json` 中
- 重启机器人后未触发的提醒会自动重新调度

## Bilibili 视频下载功能

机器人自动检测并下载 Bilibili 视频，支持多种链接格式。

### 支持链接格式
- `https://www.bilibili.com/video/BVxxx`
- `https://b23.tv/xxx`（短链接）
- `https://www.bilibili.com/video/avxxx`（旧版AV号）
- `https://www.bilibili.com/bangumi/play/ssxxx`（番剧）
- `https://www.bilibili.com/bangumi/play/epxxx`（剧集）

### 使用方法
1. 在私聊或群聊中发送任意支持的B站链接
2. 机器人会自动检测并开始下载
3. 下载完成后发送视频文件到当前聊天窗口

### 注意事项
- **下载需要时间**：视频下载和转码需要一定时间，请耐心等待
- **文件大小限制**：QQ对发送的视频文件大小有限制（通常≤100MB）
- **FFmpeg推荐**：建议安装 FFmpeg 以获得更好的视频兼容性
- **Cookies（可选）**：如果需要下载高清或会员视频，可以在 `~/.yt-dlp/cookies.txt` 放置 cookies 文件
- **网络要求**：需要稳定的网络连接访问B站和下载视频

### 常见问题
#### Q：为什么下载的视频没有声音？
A：可能因为FFmpeg未安装。请安装FFmpeg并确保其在系统PATH中。

#### Q：下载失败怎么办？
A：可能原因：
1. 视频需要B站大会员权限
2. 网络连接问题
3. 视频已被删除或屏蔽
4. FFmpeg相关错误（尝试安装FFmpeg）

#### Q：可以下载多长的视频？
A：受QQ文件大小限制，建议下载较短的视频（≤10分钟）。

## 常见问题

### Q1：为什么更新 prompt 后历史被清除了？
A：这是设计行为。为了避免旧的对话历史干扰新的角色设定，更新 prompt 时会自动清除历史。如果需要保留历史，请先使用 `/history` 导出重要信息。

### Q2：命令在群聊中如何使用？
A：在群聊中，需要 @机器人 后发送命令，例如：`@MarsAI /help`

### Q3：对话历史保存在哪里？
A：所有会话历史都保存在 `session/user_sessions.json` 文件中。私聊和群聊的历史分开存储。

### Q4：最多可以保存多少条历史记录？
A：每个会话最多保存 200 条最新的消息，旧消息会被自动删除。

### Q5：如何完全重置所有设置？
A：删除 `session/` 目录下的所有文件，然后重启机器人。

### Q6：为什么 `/summary` 有时会失败？
A：可能的原因：
1. 没有对话历史
2. API 调用失败
3. 网络连接问题
4. 模型暂时不可用

## 文件结构
- `session/user_sessions.json` - 存储所有会话历史
- `session/bot_config.json` - 存储系统配置（模型和 prompt）
- `src/plugins/mars_ai/__init__.py` - AI 命令实现代码
- `src/plugins/bilibili_downloader/__init__.py` - B站视频下载器插件
